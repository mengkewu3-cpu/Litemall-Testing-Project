这份笔记浓缩了你今天掌握的所有核心行业黑话，也是你应对初级面试的极佳素材。

1. 核心理论扫盲区
黑盒测试：把软件当成看不见内部的黑盒子，只关心输入和输出。重点测试功能逻辑和用户体验（点点点）。

白盒测试：盒子是透明的，深入代码底层，检查程序的逻辑路径和 if-else 分支（通常由开发或测开完成）。

灰盒测试：介于两者之间，既看前端界面，也看后台接口响应和数据库状态（如使用 Postman 测接口）。

2. 测试用例设计“两大护法”
等价类划分法：将无限的输入分为“有效”和“无效”。

实战：测试密码，正确的密码是有效等价类；空密码、错误密码是无效等价类。

边界值分析法：Bug 最容易出现在边界上！

实战：规定密码 6-12 位。一定要测 5位、6位、12位、13位。

3. 缺陷管理（提 Bug 的艺术）
一个能让开发无话可说的标准 Bug 单必须包含：

Bug 标题：【模块名】+ 操作步骤 + 实际结果（例如：【登录模块】输入5位密码点击登录，页面报500错误）。

严重程度：致命、严重、一般、轻微。

测试环境：如 Windows 10, Chrome 115 版本。

复现步骤、预期结果、实际结果。

证明附件：报错截图或录屏。

4. 面试必考：GET 与 POST 的区别
GET 偏向于“获取”数据，像寄明信片，参数直接暴露在 URL 链接后面，安全性相对较低，传输数据量小。

POST 偏向于“提交”数据，像寄带锁包裹，参数藏在请求体（Body）里，相对更安全，可传输海量数据。

5. 接口测试实战（Postman 核心 4 步）
选择请求方法（Method）：下拉选择 GET 或 POST。

填写请求地址（URL）：例如 https://httpbin.org/post。

配置请求体（Body）：选择 raw -> JSON，严格按照带逗号的英文 JSON 格式写入数据（例如：{"username": "test", "password": "123"}）。

发送与断言：点击 Send，查看下方状态码（200 OK 代表成功，500 代表服务器报错，400 代表你发的数据格式有误）。
